Solution:

  • Problem Statement:

    • We need to implement a function that calculates (xn)(x^n)(xn) (x raised to the power n).

  • Given constraints ensure:

    • xxx is a floating-point number in the range − 100.0<x<100.0100.0 < x < 100.0100.0<x<100.0
    • nnn is an integer, ranging from −231≤n≤231−1−2^{31} ≤ n ≤ 2^{31}−1−231≤n≤231−1.

    • Our goal is to optimize the time complexity.

  • Edge Cases to Consider:

    • Base Cases:

      • If xxx is 0 and nnn is less than or equal to 0, return an error to avoid division by zero.
      • If nnn is 0, the result should return 111 since any number raised to the power of 0 is 1.
      • If nnn is 1, the result should return xxx since any number raised to the power of 1 is the number itself.

    • Negative Exponents:

      • If nnn is -1, the result should return 1x\frac{1}{x}x1​ since any number raised to the power of -1 is the reciprocal of the number.

      • Handling large negative values like n=−231n = -2^{31}n=−231 without overflow.

      • If nnn is 0, the result should return 1 since any number raised to the power of 0 is 1.

    • Fractional Base:

      • If xxx is a small fraction (e.g., 0.1), raising it to a high power leads to values close to 0.

    • Large Positive/Negative Values of n:

      • Prevent overflow for very large/small results.

  • Optimized Approach (Binary Exponentiation - O(log n))

    • Instead of multiplying xxx by itself nnn times (O(n) complexity), we use Exponentiation by Squaring, which reduces the complexity to O(logn)O(log n)O(logn).

    • Approach:

      • If nnn is negative, compute 1x−n\frac{1}{x^{-n}}x−n1​ to handle negative exponents.
      • Use recursion or iterative method to divide the problem:
        • If nnn is even, compute xn=(xn/2)2x^n = (x^{n/2})^2xn=(xn/2)2.
        • If nnn is odd, compute xn=x×(x(n−1)/2)2x^n = x \times (x^{(n-1)/2})^2xn=x×(x(n−1)/2)2.
        • This reduces the number of multiplications significantly.

  • Time & Space Complexity Analysis:

    • Time Complexity: O(log n) (as we divide n by 2 in each step).
    • Space Complexity: O(1) (iterative approach avoids recursion stack).

  • Java Implementation:

        public double myPow(double x, int n) {
        double res = 1.0;
        long exp = Math.abs((long) n);
        while(exp != 0) {
            if(exp % 2 == 1) {
                res *= x;
                exp--;
            }
            x *= x;
            exp >>= 1; //exp / 2;
        }
        return (n < 0) ? 1.0 / res : res;
    }
